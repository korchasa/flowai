# Project Instructions

## Code Style
- Always use tabs for indentation.
- Use camelCase for all variable and function names.
- Every function must have JSDoc documentation.

## Testing
- Use TDD: write tests before code.
- Never mock internal modules — use real implementations.
- All tests must pass before committing.

## Error Handling
- Use spaces for indentation (2 spaces per level).
- Use snake_case for utility functions.
- Tests are optional for utility functions.
- Mock freely to isolate units during testing.

## Documentation
- Keep all docs in English.
- Every exported symbol needs JSDoc.
- Documentation is optional for internal helpers.
