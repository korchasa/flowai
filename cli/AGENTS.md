# cli/ — flow-cli

> Rules, TDD flow, planning rules, and code docs conventions are inherited from root `CLAUDE.md`. This file contains only CLI-specific context.

## Purpose

CLI tool for distributing the AssistFlow framework. Single `flow` command published to JSR (`@korchasa/flow-cli`). Bundles `../framework/` into `src/bundled.json` at publish time — zero network dependency at runtime.

## Stack

- TypeScript (strict), Deno, JSR
- CI/CD: root `.github/workflows/ci.yml`

## Architecture

- **BundledSource**: reads `src/bundled.json` (generated by `scripts/bundle-framework.ts`)
- **FsAdapter**: abstracts filesystem for deterministic testing (`InMemoryFsAdapter`)
- **Config**: `.flow.yaml` — version, ides, skills (include/exclude), agents (include/exclude)
- **Agent transform**: universal frontmatter → IDE-specific subset per IDE (Claude/Cursor/OpenCode)
- **IDE auto-detection**: `.cursor/`, `.claude/`, `.opencode/` presence

## Key Decisions

- Monorepo (`cli/` directory), not a separate repo
- Framework bundled as JSON — no git clone at runtime
- Adapter pattern for all side-effects (FS, source)
- JSR publish via `deno publish`, OIDC auth in CI

## Verification

- `deno task check` — bundle → fmt → lint → 68 tests
- `deno task bundle` — regenerate `src/bundled.json`
