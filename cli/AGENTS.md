# cli/ — flowai

> **Inheritance:** All rules from root `AGENTS.md` (TDD flow, planning, code docs, compressed style) apply here automatically. This file adds only CLI-specific context. When a rule here conflicts with root, this file wins for `cli/` scope.

## Purpose

CLI tool for distributing the flowai framework. Single `flowai` command published to JSR (`@korchasa/flowai`). Bundles `../framework/` into `src/bundled.json` at publish time — zero network dependency at runtime.

## Stack

- TypeScript (strict), Deno, JSR
- CI/CD: root `.github/workflows/ci.yml`

## Architecture

- **BundledSource**: reads `src/bundled.json` (generated by `scripts/bundle-framework.ts`)
- **FsAdapter**: abstracts filesystem for deterministic testing (`InMemoryFsAdapter`)
- **Config**: `.flowai.yaml` — version, ides, skills (include/exclude), commands (include/exclude), agents (include/exclude)
- **Agent transform**: universal frontmatter → IDE-specific subset per IDE (Claude/Cursor/OpenCode)
- **IDE auto-detection**: `.cursor/`, `.claude/`, `.opencode/` presence

## Generated Files (gitignored)

- `src/bundled.json` — produced by `deno task bundle` from `../framework/`. NEVER stage manually.
- `src/_version.ts` — produced by the same task from root `deno.json` version.
- Both fail `git ls-files --error-unmatch` by design — that is not a bug.
- `cli/src/main.ts` references both at import time; running `deno check cli/src/main.ts` after a fresh clone requires `deno task bundle` first.

## Test Execution Quirks

- All CLI tests require `-A` (read+write+run+net+env). `deno test src/...` without `-A` fails on permission errors that look like real bugs.
- Some integration tests use **relative path** `cli/src/main.ts` (e.g. `ide_test.ts::CLI - --version`). Run them ONLY from repo root, not from `cli/` subdir — otherwise the path becomes `cli/cli/src/main.ts` and the test fails with `Module not found`.

## Key Decisions

- Monorepo (`cli/` directory), not a separate repo
- Framework bundled as JSON — no git clone at runtime
- Adapter pattern for all side-effects (FS, source)
- JSR publish via `deno publish`, OIDC auth in CI
- Commands and skills both install into `.{ide}/skills/`; the `disable-model-invocation: true` flag on commands is injected by `injectDisableModelInvocation` in `src/sync.ts` at sync time, NOT authored in source. Validator (`scripts/check-skills.ts`) enforces this invariant on `framework/<pack>/{commands,skills}/*/SKILL.md`.

## Verification

- `deno task check` — bundle → fmt → lint → tests + validators. Run from repo root.
- `deno task bundle` — regenerate `src/bundled.json` and `src/_version.ts`.
